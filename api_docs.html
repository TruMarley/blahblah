API Reference

The RingDesk API is a REST API that allows you to interact with your AI receptionist programmatically. All responses are JSON unless otherwise noted. The API runs on your self-hosted server instance.

Authentication is not required for local development. For production deployments, set the RINGDESK_API_KEY environment variable to enable bearer token authentication.

Base URL  http://localhost:5050
Health
Service status and configuration verification.
GET /api/health Service health check ▼
Description
Returns the current health status of the RingDesk service, including the shop name, AI agent name, and server timestamp. Use this endpoint to verify your server is running correctly after deployment.
Response 200
{
  "status": "ok",
  "service": "Crown & Blade Barbershop",
  "ai": "Alex",
  "timestamp": "2026-04-16T23:00:00.000000"
}
Voice / Call
Twilio webhook endpoints and voice agent controls. The /incoming-call endpoint is called by Twilio on every inbound call.
POST /incoming-call Twilio inbound call webhook ▼
Description
Called by Twilio when a call arrives at your phone number. Returns TwiML that connects the call to the AI voice agent via a WebSocket Media Stream. Configure this URL in your Twilio phone number's "Voice & Fax" webhook settings.
Twilio Request Body (form-encoded)
CallSid=CAxxxxxxxx
From=%2B17865550100
To=%2B17865559999
CallStatus=ringing
Response — TwiML
<?xml version="1.0" encoding="UTF-8"?>
<Response>
  <Connect>
    <Stream url="wss://your-server.com/ws/audio" />
  </Connect>
</Response>
GET /api/demo Return mock demo data ▼
Description
Returns mock appointment and call data for the demo dashboard. Used by the /dashboard demo mode to show realistic data without requiring live calls.
Response 200
{
  "appointments": [
    {
      "name": "Marcus J.",
      "service": "Fade",
      "time": "10:30 AM",
      "status": "confirmed"
    }
  ],
  "missed_calls": 3,
  "callbacks": 1
}
POST /api/voice-preview Generate TTS audio preview ▼
Description
Calls Deepgram Aura TTS and returns a WAV audio file for browser playback. Used by the voice settings page to preview voices before saving. Requires DEEPGRAM_API_KEY environment variable.
Request Body
{
  "text": "Thanks for calling Crown and Blade. I'm Alex. How can I help?",
  "voice_id": "aura-asteria-en"
}
Response
Returns audio/wav binary data. Play with new Audio(URL.createObjectURL(blob)).
Appointments & Analytics
Read appointment data, analytics, callbacks, and send reminders.
GET /api/stats Dashboard KPI summary ▼
Description
Returns today's appointment count, missed calls, callback requests, and SMS confirmations sent. Used by the owner dashboard.
Response 200
{
  "appointments_today": 8,
  "missed_calls": 2,
  "callbacks_pending": 1,
  "sms_sent_today": 8
}
GET /api/analytics Full analytics dataset ▼
Description
Returns 30-day booking trends, 7-day hourly peak data, service breakdown percentages, and answered vs. missed call ratio. Feeds the analytics dashboard charts.
Response 200
{
  "bookings_30d": [12, 18, 15, /* 30 days */],
  "peak_hours": { "10": 24, "11": 31 },
  "services": [
    { "name": "Haircut", "count": 45, "pct": 42 }
  ],
  "answered": 312,
  "missed": 28
}
POST /api/callbacks/{cb_id}/resolve Mark callback resolved ▼
Description
Marks a pending callback request as resolved. The cb_id is the SQLite row ID from the callbacks table, returned in the /api/stats response.
Response 200
{ "ok": true }
POST /api/send-reminder Manually trigger SMS reminder ▼
Description
Manually sends a reminder SMS to a specific phone number. Normally reminders are sent automatically 24h and 2h before appointments.
Request Body
{
  "phone": "+17865550100",
  "message": "Reminder: Your appointment is in 2 hours!"
}
Response 200
{ "ok": true, "sent": true }

Webhooks
Inbound webhooks from Twilio and other services.
POST /api/webhook/twilio-status Twilio SMS delivery status ▼
Description
Called by Twilio when an SMS delivery status changes. Configure this as the "Status Callback URL" in your Twilio messaging service. Logs each event to sms_log.json.
Twilio Request (form-encoded)
MessageSid=SMxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
MessageStatus=delivered
To=%2B17865550100
From=%2B17865559999
Response 200
{ "ok": true }
GET /api/sms-log Recent SMS delivery events ▼
Description
Returns the last 50 SMS delivery status events received from Twilio webhooks.
Response 200
{
  "events": [
    {
      "MessageSid": "SMxxxxxxxx",
      "MessageStatus": "delivered",
      "To": "+17865550100",
      "timestamp": "2026-04-16T23:15:00"
    }
  ],
  "total": 1
}
